# Prototipos
Los modelos de prototipos del Sistema Integrado de Gestión Fronteriza son representaciones visuales diseñadas para validar los requerimientos y la usabilidad de la plataforma antes de su programación. Para garantizar que el sistema prevenga errores y sea fácil de usar, ambos modelos fueron evaluados utilizando las 10 Heurísticas de Nielsen. El sistema se divide en dos enfoques integrados:  Portal de Tránsito Fronterizo (Turistas): Es una plataforma móvil de autoatención diseñada paso a paso para guiar al usuario fácilmente. Tras iniciar sesión con Clave Única, el viajero ingresa los datos de su vehículo[cite: 7, 8], sube su documentación (con un límite de 5 MB por archivo para evitar saturar la base de datos)[cite: 3, 7, 8], registra a los pasajeros para la revisión de la PDI[cite: 5, 7, 8] y responde una declaración jurada dinámica del SAG[cite: 5, 7, 8]. Al finalizar, el sistema consolida la información y emite un Código QR válido por 24 horas para presentar en la frontera[cite: 5, 7, 8].  Portal Interno (Funcionarios): Es una interfaz de alta velocidad optimizada para el control aduanero. Mediante un acceso institucional seguro[cite: 7, 8], el funcionario utiliza la cámara para escanear el Código QR del turista[cite: 5, 7, 8]. La pantalla principal es un panel visual tipo "semáforo" que muestra en tiempo real si el cruce está aprobado por la PDI, el SAG y Aduanas. Con esta información a la vista, el funcionario puede emitir la resolución y levantar la barrera con un solo clic
